3.1 SYSTEM OVERVIEW
================================================================================

The Keycard Access Control & Monitoring System is a comprehensive solution designed 
to manage apartment access through RFID keycards while providing real-time camera 
surveillance capabilities. The system integrates hardware components (ESP32 camera 
module, RFID readers), cloud services (Firebase Firestore for data management), and 
web applications (landlord and tenant dashboards) to create a unified property 
management platform. The architecture employs a three-tier design: frontend web 
applications for user interaction, a Python backend server for stream proxying and 
API management, and Firebase cloud services for data persistence and real-time 
synchronization.

Description: The Level 1 DFD breaks down the system into two primary processes. 
The tenant/landlord access flow captures RFID data from keycards or mobile devices, 
validates them against the database, and either grants access by unlocking the door 
or logs the failed attempt. The camera monitoring flow streams video from the ESP32 
camera through a Python proxy server, applies PIN verification security, and sends 
real-time notifications to relevant users through Firebase when access is detected.

================================================================================
CHAPTER 4: SYSTEM FEATURES & USER INTERFACE
================================================================================

4.1 SYSTEM FEATURES OVERVIEW
================================================================================

The Keycard Access Control & Monitoring System provides comprehensive features 
organized across two primary user interfaces (Landlord Dashboard and Tenant Dashboard) 
with supporting infrastructure for RFID processing, camera streaming, and data 
management. All features integrate Firebase Firestore for real-time data synchronization 
and employ security mechanisms including PIN verification for camera access.

FEATURE 13: REAL-TIME CAMERA STREAM PROXY
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
Screenshot Recommendation: No visual screenshot needed - describe architecture in 
system documentation.

Feature Description: The Python-based camera stream proxy (`/api/camera/stream` endpoint) 
fetches MJPEG video from the ESP32 camera module and serves it through the local 
server, eliminating direct IP exposure and improving security. The proxy automatically 
attempts multiple fallback endpoints (/capture, /stream, /jpg, /mjpeg) to ensure 
compatibility with various ESP32 firmware versions. The stream is accessed via 
localhost:8000, providing centralized control and bandwidth management.

Feature Description: Firebase Firestore integrates with the system to provide real-time 
data synchronization across all connected clients. When a tenant receives approval for 
a request or when a landlord initiates camera monitoring, Firestore instantly propagates 
these updates to active user sessions. This event-driven architecture ensures zero-delay 
updates without requiring page refreshes, critical for notification systems and access 
control updates.

FEATURE 16: PIN-PROTECTED CAMERA ACCESS
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€

Feature Description: Camera access requires separate PIN verification (Landlord PIN:

monitoring. Failed PIN attempts are logged and rate-limited to prevent brute-force 
attacks. PIN entry uses secure password input masking for visual privacy during entry.

Feature Description: When landlords access tenant camera feeds, Firebase automatically 
notifies the monitored tenant in real-time with landlord name and access start time. 
This transparency mechanism establishes trust and ensures tenants are aware of 
surveillance, meeting privacy law requirements. Notifications remain visible in the 
tenant interface throughout the monitoring session.
