üìù Overview

The Context Diagram (DFD Level 0) shows the SecurAccess system as a single process box interacting with six external entities. This high-level view represents the entire system's boundary and all external interactions.

üë• External Entities

Entity Type Description Key Interactions
Tenant Actor Residential user who needs access to their apartment Scan RFID card, View access history, Manage profile
Landlord/Admin Actor Property manager who oversees all residents Approve requests, Monitor access, Manage permissions
Firebase Cloud Data Store Cloud-based database for persistent data storage Store records, Retrieve user data, Real-time sync
RFID Reader Device IoT sensor that reads keycard data Send card IDs, Receive unlock commands
Electronic Lock Device Servo-controlled door lock mechanism Receive unlock signal, Unlock door for 10 seconds
Camera Device ESP32 camera module for live monitoring Stream video feed, Send MJPEG data

üîÑ Data Flows

Tenant Interactions

  • Login Credentials: Tenant ‚Üí System (email, password)
  • Dashboard Display: System ‚Üí Tenant (access history, statistics)
  • Profile Update Request: Tenant ‚Üí System (updated information)
  • Access Records: System ‚Üí Tenant (RFID log with timestamps)

Landlord Interactions

  • Admin Login: Landlord ‚Üí System (credentials)
  • Approval Decisions: Landlord ‚Üí System (approve/reject requests)
  • Camera Commands: Landlord ‚Üí System (select apartment, request stream)
  • Permission Changes: Landlord ‚Üí System (grant/revoke access)
  • System Reports: System ‚Üí Landlord (all logs, statistics)

Hardware Interactions

  • RFID Scan: Reader ‚Üí System (card ID, timestamp)
  • Verify Response: System ‚Üí Reader (confirmation)
  • Unlock Command: System ‚Üí Lock (servo rotation signal)
  • Lock Status: Lock ‚Üí System (locked/unlocked state)
  • Camera Stream: Camera ‚Üí System (MJPEG video data)
  • Stream Request: System ‚Üí Camera (connect, authenticate)

Database Interactions

  • Store Access Log: System ‚Üí Firebase (RFID scan records)
  • Retrieve User Data: System ‚Üí Firebase (tenant/admin info)
  • Update Database: System ‚Üí Firebase (change approvals)
  • Real-time Sync: Firebase ‚Üí System (listen for changes)

‚öôÔ∏è System Processes (Expanded at Level 1)

The following processes are shown as a single box at Level 0 but expanded in DFD Level 1:
  1. User Authentication (login, verify credentials)
  2. Request Management (approve/reject changes)
  3. RFID Processing (verify card, check permissions)
  4. Lock Control (activate servo, monitor status)
  5. Camera Access (verify permissions, stream video)
  6. Data Management (store, retrieve, sync records)
